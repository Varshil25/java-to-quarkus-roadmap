// JDBC Practical
	4. Load and Register the Driver
		Class.forName("ClassName"); ClassName is "com.mysql.cj.jdbc.Driver".
		// You must throw or catch these exceptions because Class.forName() and DriverManager.getConnection() are declared with checked exceptions in the Java API. The compiler will refuse to compile your code if you ignore them.
		
		-> must throws the ClassNotFoundException -- Java tried to load a class by its name (as a String), but could not find it on the classpath.
		
	5. Establish the Connection:
		Invoking the getConnection request form the Driver Class.
		
		--> must Throws the SQLException -- Something went wrong while Java was trying to communicate with the database.
		e.g., DriverManager.getConnection(url, user, password);
		
	6. Statement Object in JDBC.
		A Statement object is created because you need a "messenger" to send your SQL queries from Java to the Database.
	
		--> After a connection you must set the statement that only way to run SQL Directly.
		--> Connection only open the door to the database. Connection has no method to run a SQL Quearies.

		1. What is st.execute()?

🔑 One Line Summaries

	Class.forName() = Load driver by name
	Connection = Open door to DB
	Statement = Send SQL to DB
	ResultSet = Hold SELECT results
	execute() = Universal SQL runner
	finally = Always closes resources
	JDBCUtil = Reusable DB helper class
	

Prepared Statement
	--> Use PreparedStatement always when your SQL has dynamic values coming from user input — it is safer, cleaner, and faster than Statement.
		
		
Statement V/S PrepareStatement :
    --> Use Statement for fixed SQL with no user input. Use PreparedStatement always when values come from the user.
    
    
🔑 Golden Rules

Always put WHERE condition ? last in UPDATE
Always use executeUpdate() for INSERT, UPDATE, DELETE
executeUpdate() returns rows affected — check it!
Always close PreparedStatement in finally block

	// Insert
	String sql = "INSERT INTO studentinfo(id, sname, sage, scity) VALUES(?,?,?,?)";
	pst = connect.prepareStatement(sql);
	
	pst.setInt(1, 101);
	pst.setString(2, "John");
	pst.setInt(3, 22);
	pst.setString(4, "Boston");
	
	int rowAffected = pst.executeUpdate();
	System.out.println("Inserted: " + rowAffected + " row");
	
	
	
	// Update
	String sql = "UPDATE studentinfo SET sname=?, sage=?, scity=? WHERE id=?";
	pst = connect.prepareStatement(sql);
	
	pst.setString(1, "Tisha");
	pst.setInt(2, 25);
	pst.setString(3, "NewYork");
	pst.setInt(4, 101);        // ← WHERE condition always LAST
	
	int rowAffected = pst.executeUpdate();
	System.out.println("Updated: " + rowAffected + " row");	
	
	
	
	// Delete
	String sql = "DELETE FROM studentinfo WHERE id=?";
	pst = connect.prepareStatement(sql);
	
	pst.setInt(1, 101);        // ← just the WHERE value
	
	int rowAffected = pst.executeUpdate();
	System.out.println("Deleted: " + rowAffected + " row");




🔑 Key Rules

Use executeQuery() for SELECT — never executeUpdate()
rs.next() returns false if no data found
Use while(rs.next()) for multiple rows
Use if(rs.next()) for single row
Always close ResultSet in finally block

Retrieve — With WHERE Condition
	String sql = "SELECT * FROM studentinfo WHERE id=?";
	pst = connect.prepareStatement(sql);
	
	pst.setInt(1, 101); // ← set the WHERE value
	
	ResultSet rs = pst.executeQuery();
	
	if(rs.next()) {
	    System.out.println(rs.getInt("id") + " " + rs.getString("sname") + " " + rs.getInt("sage") + " " + rs.getString("scity"));
	} else {
	    System.out.println("No record found!");
